## 一、报价规则(已确认的计算模型)

### 1. 袋型与展开面积(2026-07-22 调整:普通袋细分为四种,异形改为工艺)

| 袋型 | 输入尺寸 | 展开面积公式 |
|---|---|---|
| 三边封袋 | 宽 W、高 H | `W × H × 2` |
| 自立袋 | 宽 W、高 H、底风琴 D | `W × (H + D) × 2` |
| 风琴袋 | 宽 W、高 H、侧风琴 G | `(W + G) × H × 2` |
| 中封袋 | 宽 W、高 H | `W × H × 2` |
| 八边封袋 | 宽 W、高 H、侧风琴 G | `(W + G) × H × 2 + W × G` |

选"异形"工艺时,尺寸按外接矩形填写,套用所选袋型公式。
面积换算:cm² ÷ 10000 = ㎡。总印刷面积 = 单袋展开面积 × 数量。

> 公式为行业默认版;风琴袋、中封袋公式为拆分时新增,上线后请重点校对。公式是配置项,有偏差随时调。

### 2. 费用构成

**总价 = round( (印刷费 + 复合制袋费 + 加价项费用) × 折扣 )**,四舍五入到元。

#### 印刷费
- 600 元/㎡,低消 100 元:`印刷费 = max(600 × 总印刷面积㎡, 100)`

#### 复合制袋费(元/个)

| 袋型 | 单价 |
|---|---|
| 三边封袋 / 自立袋 / 风琴袋 / 中封袋 | 30 |
| 八边封袋 | 60 |

异形不再是独立袋型,改为工艺项,在制袋费基础上**叠加**:普通袋系 +60 元/个、八边封 +90 元/个(即原规则"异型普通袋加60""异型八边封加90"中的金额就是异形叠加价;异形普通袋合计制袋 30+60=90,异形八边封 60+90=150)。

#### 表面选项(2026-07-22 补充规则)

- 袋子整体表面:**亮面 / 哑面** 二选一(必选,本身不加价)。
- 选了开窗时,开窗处表面同样 **亮面 / 哑面** 二选一。
- **开窗处表面与袋子整体表面不一致时,每个袋子另加 80 元**(线上直接算价;取代原"开窗处亮面线下人工报价"规则)。

#### 加价项(元/个)

| 工艺 | 普通袋系 | 八边封袋 | 说明 |
|---|---|---|---|
| 异形 | +60 | +90 | 与制袋费叠加;尺寸按外接矩形计 |
| 牛皮纸开窗 | 60 | 90 | |
| 开窗处表面与整体不一致 | 80 | 80 | 见上方表面选项规则 |
| 拉链 | 0 | 0 | **不另加价**;仅三边封袋 / 自立袋 / 八边封袋可选 |
| 嘴 | 25 | 25 | **3 个起**:不足 3 个按 3 个计费(低消 75 元)。已改为线上算价,取代原"嘴袋人工报价" |
| 气阀 | 10 | 10 | **2 个起**:不足 2 个按 2 个计费(低消 20 元) |

### 3. 数量折扣(按"单款"计算,作用于全部费用)

| 单款数量 | 折扣 |
|---|---|
| 1–4 个 | 不打折 |
| 5–9 个 | 9 折 |
| 10–19 个 | 8 折 |
| 20–49 个 | 7 折 |
| 50–99 个 | 6 折 |
| **≥ 100 个** | **5 折**(100 及以上统一) |

### 4. 特小袋规则(已确认)

- **特小袋定义**:单个袋子印刷费 < 11 元(即单袋展开面积 < 11/600 ≈ 0.0183㎡)。
- **规则**:特小袋在(折前)总印刷费低于 100 元时,**整单不打任何折扣**;总印刷费 ≥ 100 元后各档折扣恢复正常。
- 实现:`if (单袋印刷费 < 11 && 面积×600×数量 < 100) 折扣 = 1`。

### 5. 其他已确认事项

- **金额取整**:折后总价四舍五入到整元。
- **报价文案**:复制文本末尾附默认说明——"以上为打样费用,不含税运,最终以实际确认为准。"(文案可配置)
- **人工报价项**:当前规则下已全部线上算价,暂无人工报价项(开窗亮面 → 表面不一致 +80 元/个;嘴袋 → 嘴加价 25 元/个)。引擎保留"人工报价"状态能力,以备后续新增特殊工艺。

## 三、产品形态

### 阶段一:网页版(移动优先)

单页应用,操作流程:

```
选袋型 → 输尺寸(自动算展开面积)→ 选表面(亮/哑)→ 选工艺(开窗及开窗处表面/嘴/气阀)→ 输数量
                    ↓
        实时显示报价结果(2026-07-23 起对客户隐藏计算参数):
        · 小计(折前)
        · 折扣(命中哪一档;特小袋不打折时说明原因)
        · 应收总价(四舍五入到元)
                    ↓
        [复制报价文本] 一键复制,直接粘贴发微信给客户
```

关键功能点:

- **多款合并报价**:一次打样常有多款,支持"添加一款",折扣按单款各自计算,底部汇总总价。
- **人工报价拦截**:选中人工报价工艺时不出价格,显示"该工艺需人工报价,请联系客服"。
- **对客隐藏参数**:展开面积、印刷单价、各项费用等计算参数不在页面和报价文本中出现(2026-07-23 需求);引擎内部仍完整计算并保留明细数据,以后要做"仅业务员可见的内部明细"开关很容易。
- **规则可配置**:单价、折扣档、低消、面积公式、文案全部集中在配置文件(`rules.ts`),改价不动逻辑代码。

### 阶段二:微信小程序

- **技术路线:uni-app(Vue3)**——一套代码同时产出 H5 网页版和微信小程序。
- 报价引擎为**独立纯 TS 模块**,网页和小程序共用,保证两端价格一致。
- 小程序增值功能(可选):报价单分享图、客服消息一键咨询、询价记录。

### 阶段三(可选):后台与云端

- "改价不发版":规则配置放云端(微信云开发/serverless),启动时拉取。
- 询价留资、报价历史。

## 四、技术方案

| 层 | 选型 | 理由 |
|---|---|---|
| 报价引擎 | 纯 TypeScript 模块 + 单元测试 | 框架无关,三端复用;价格计算必须有测试兜底 |
| 网页版 | Vite + Vue3 | 与 uni-app 同为 Vue 生态,迁移成本最低 |
| 小程序 | uni-app | 一套代码多端 |
| 部署 | 静态托管(无需服务器) | 阶段一纯前端,零运维成本 |
| 金额处理 | 以"分"为整数运算,最终四舍五入到元 | 避免浮点误差 |

### 报价引擎接口草案

```ts
interface QuoteInput {
  bagType: '三边封袋' | '自立袋' | '风琴袋' | '中封袋' | '八边封袋';
  widthCm: number;          // 宽(异形时为外接矩形宽)
  heightCm: number;         // 高
  gussetCm: number;         // 自立袋=底风琴;风琴袋/八边封=侧风琴;其余不用
  quantity: number;         // 单款数量
  surface: '亮面' | '哑面'; // 袋子整体表面
  shaped: boolean;          // 异形工艺,+30元/个叠加
  window: boolean;          // 牛皮纸开窗
  windowSurface: '亮面' | '哑面'; // 开窗处表面;与整体不一致时 +80元/个
  zipper: boolean;          // 拉链,0元;仅三边封/自立/八边封可选
  spout: boolean;           // 嘴 25元/个,不足3个按3个计
  valve: boolean;           // 气阀 10元/个,不足2个按2个计
}

interface QuoteResult {
  status: 'ok' | 'manual';        // manual = 需人工报价
  areaPerBagM2: number;           // 单袋展开面积
  printingFee: number;            // 印刷费(已含低消逻辑)
  minChargeApplied: boolean;      // 是否触发100元低消
  bagMakingFee: number;           // 复合制袋费
  optionFees: { name: string; fee: number }[];
  discountRate: number;           // 1 / 0.9 / 0.8 / 0.7 / 0.6 / 0.5
  discountTier: string;           // 例如 "10-19个 8折" / "特小袋低消内不打折"
  total: number;                  // 折后四舍五入到元
  breakdown: string[];            // 明细文本行,用于展示和复制
}
```
